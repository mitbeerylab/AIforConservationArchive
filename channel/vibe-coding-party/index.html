"""
In our identify page we have an ecosystem for labeling images - we want to add a bounding box feature so people have to draw a box around what they see in the image - and then when submitting the observation, we need to submit the bounding box coordinates as well.
Please check the full repo & make sure to update models, features, and routes as necessary
"""

"""
ok, this is looking great -
let's make sure the bbox attributes match the schema at: https://camtrap-dp.tdwg.org/data/
Also, there might be more than one species in the photo - so instead of generally drawing a bounding box, let's make it so each species picked has a "draw box" icon and tooltip next to it, so the person identifying the species can draw one or more boxes around each animal - if they select 1 animal, one box must be drawn - if they select 2 animals of a species then 2 boxes can be drawn, and all combinations of that idea.
the icon for draw boxes should indicate what color box outline is used for each species
"""

ok, another day & some more experiences with 🌠 vibes 🤩

1 - This was big. Our data is in MongoDB to run the website - but our eventual goal that nobody really wanted to take on is : get the data into CamtrapDP format.

  • I downloaded example datasets and schema from camtrap dp into a repo

  • I asked chatGPT to structure a script for me to download an example from our MongoDB

  • I casually asked Jules to take a look at my target schema and current schema and build a tool to process the files.

looking at the database files in the mongo_exports folder saved as samples and schema json files -- please write some python functions to convert what you find to camtrap dp format.
camtrap dp format is defined in camtrap_standards/definitions
there are example csv files in camtrap_standards/examples
The goal is to be able to download all of the data in the mongo database and format it into csv files with the information that camtrap dp specifies.
try to get as much data as possible, but if there are missing fields, it's ok to leave them empty.

IT WORKED. It included a lot of reformatting and edge cases and even built a testing dataset. Months and months of time saved - incredible.
