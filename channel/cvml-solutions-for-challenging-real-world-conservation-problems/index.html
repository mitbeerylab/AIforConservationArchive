wildlabs.net
Can deep learning identify seabirds? (species, within-species, individual) | WILDLABS
Hello everyone,I am looking for insight on the feasibility of identifying seabirds from photos taken at sea (such as these <https://www.flickr.com/photos/9191812@N02/albums/72157708853397813>)  to different levels of precision:1) species level. This can already be tested, I think, as it is already implemented in camera trap pictures.2) phenotype level within species. In some species (for example, the Northern Fulmar <https://www.flickr.com/search/?text=northern%20fulmar>), a continuous phenotypic gradient exists (for example, from darkest to lightest plumage). I suspect that the difficulty would be in assigning a score - though one end of the gradient could be considered as the typical plumage (100% ID score) and the other end as the aberrant plumage (0% ID score), with scores in between giving a proxy for phenotype.  3) individual level. Like the BearID Project is able to identify individual bears using face recognition techniques, could deep learning identify individual birds using not only face geometry but also plumage patterns (for example, face, under- and upper-wing patterns as in the Black-capped Petrel <https://www.flickr.com/search/?text=black-capped%20petrel>)? Using geo-referenced images, this kind of analyses would help look into species/phenotype distribution. Individual identification could also be used to estimate population sizes (a conservation metric that is missing in several endangered pelagic seabirds).
Original URL: https://www.wildlabs.net/discussion/can-deep-learning-identify-seabirds-species-within-species-individual

arXiv.org
Transferring Dense Pose to Proximal Animal Classes
Recent contributions have demonstrated that it is possible to recognize the<br>pose of humans densely and accurately given a large dataset of poses annotated<br>in detail. In principle, the same approach could be extended to any animal<br>class, but the effort required for collecting new annotations for each case<br>makes this strategy impractical, despite important applications in natural<br>conservation, science and business. We show that, at least for proximal animal<br>classes such as chimpanzees, it is possible to transfer the knowledge existing<br>in dense pose recognition for humans, as well as in more general object<br>detectors and segmenters, to the problem of dense pose recognition in other<br>classes. We do this by (1) establishing a DensePose model for the new animal<br>which is also geometrically aligned to humans (2) introducing a multi-head<br>R-CNN architecture that facilitates transfer of multiple recognition tasks<br>between classes, (3) finding which combination of known classes can be<br>transferred most effectively to the new animal and (4) using self-calibrated<br>uncertainty heads to generate pseudo-labels graded by quality for training a<br>model for this class. We also introduce two benchmark datasets labelled in the<br>manner of DensePose for the class chimpanzee and use them to evaluate our<br>approach, showing excellent transfer learning performance.

I just recently had to deal with frame-wise prediction in videos. There's generally two Python interfaces available, OpenCV and PyAV, both piggybacking on ffmpeg and both not 100% perfect. I'd recommend OpenCV; could work as follows:

Open video and read frame-by-frame, with optional frame skip:

import cv2
import torch

reader = cv2.VideoCapture('path/to/video.mp4')

# read a frame
success, frame = reader.read()

if success:
  # convert frame to PyTorch tensor
  frame = torch.from_numpy(frame).permute((2,0,1)).float()

# to skip to the nth frame
frame_num = 5   # skip to 5th frame
reader.set(cv2.CAP_PROP_POS_FRAMES, frame_num)

# always close reader at the end
reader.release()

For the frame skip: as you can see, OpenCV allows frame-by-frame reading. If you want to have time-based reading, you need to convert the time lag to number of frames. You can extract required metadata as follows:
# get total no. frames in video
num_frames = int(reader.get(cv2.CAP_PROP_FRAME_COUNT))

# get frame rate of video
fps = reader.get(cv.CAP_PROP_FPS)

# length of video in seconds
duration = num_frames / fps

I typically use ffmpeg to extract the whole video into frames and then run python (using OpenCV or Pytorchs torchvision stack) on the individual frames:
I use this command to extract the frames to preserv the timestamps of the frames:

ffmpeg -i video.mp4 -q:v 2 -s -f image2 -frame_pts true frames_to_annotate/frame_%05d.jpg
frame_pts true implies the filename of the video encodes the frame number from the start of the video - in 1/fps seconds

could the cages be removed with a simple band-filter? run a 2-dimensional spacial FFT to create real and imaginary frequency images - the cage bars should appear as relatively stong peaks -strong enough that an algorithm could detect them automatically and compute a filter-matrix. multiply with the suppressive matrix in the frequency space, and reverse FFT back into image space and the cages should be mostly gone. alternatively a simple LPF (blurring) would probably reduce the effect of the bars significantly
finally, the frequency mask above can be expressed as a convolution - which could be learned - one could solve for a convolutional filter (aka train a CNN) that removes cage bars - if that is a frequent/common issue
